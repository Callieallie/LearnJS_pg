HTML and CSS Notes

HTML Elements

(they become tags when wrapped in <>)

  • BLOCK ELEMENTS

    • h1: The most important topic of the page (Big Newspaper Title) Try not to use more than one h1 element per page.
    • h2: Defines titles of the page's major sections (sub-sections of h1)
    • h3-h6: Each heading-type represents a further subsection of it's predecessor
    • ***Think of headers like you would the alphanumeric outline system in paper drafting
    • ***Roman Numerals, Capitalized Letters, Arabic Numerals (1-9), and Lowercase Letters
    • br: used to add a newline/carriage-return. The br element should not be used for separating paragraphs (or elements)!
      • The br element should be reserved for poems and addresses.
      • DO NOT use the br element to format a paragraph.

  • LANDMARK ELEMENTS

    • main: Represents the dominant content of the page
    • header: Contains any repeated Intro to the Page content (often the h1)
    • footer: Contains miscelanious information towards the end of a page

  • INLINE ELEMENTS

    • strong: used to strongly emphasize important sections of text in a paragraph. Is inline in nature.
      • DO NOT use a strong element as a way to bolden text.
      • HTML should be used for meaning. If you need bold text, use CSS.
      • Use strong to point out words or phrases that encompass the meaning of a paragraph or page.
    • b: "bring attention to". Draws attention but does not add emphasis.
      • Use b to add some contrast in focus.
      • Imagine the text in the b element saying "notice me :)"
    • em: "emphasis". Does not have to do with importance, but rather inflection.
      • "Did you find the keys?" emphasizes "you"
      • Again, DO NOT use em or any other HTML element for style.
    • i: "idiomatic text". Word(s) spoken in a differing voice or mood.
      • Often used for parts of text in a different language.
      • Can be used for the voice of a character in a story, or to show sarcasm.
      • The lang attribute should be put on any i element representing a language change.

  • IMAGES

    • img: image element
      • src attribute defines the source of the image (via browser or local)
      • height and width attributes define the image's aspect ratio
      • alt attribute gives the image an "alternative text" in case the image cannot be displayed
      • Providing an alternate text makes the page more accessable and better for SEO

Assorted Notes

  • emmet: a plugin used to assist in writing HTML quickly. Already installed with VS Code
  • Meaningful headings are important especially for screen readers and for SEO
  • Axe is the name of a popular accessability testing tool
  • Boilerplate: "!"+tab will create a basic HTML template
  • COMMENTS: Ctrl+"/" will create a new html comment
  • Validating HTML: validator.w3.org
